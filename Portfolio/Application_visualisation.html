Présentation du Projet

Lors de ma troisième année de BUT Science des Données (STID), j'ai participé à un projet visant à collecter des données, alimenter une base de données et construire un tableau de bord par rapport à une problématique choisie au préalable. Ce projet a été réalisé à trois sur une période de quatre semaines, et la problématique que nous avons choisie était la suivante : Quels sont les éléments qui influencent la réussite d’un film ? Le choix des technologies, de la problématique et des données utilisées était entièrement libre. Pour nous démarquer sur ce projet, nous avons décidé de réaliser une application entièrement automatisée de la collecte et de l'alimentation du tableau de bord. Pour automatiser cette application, nous avons opté pour une architecture complexe comprenant plusieurs technologies autour du cloud. Dans ce projet, j'ai principalement participé à la réalisation de l'architecture de l'application ainsi qu'à la partie scripting.

Technologies Utilisées

Pour la réalisation de ce projet, nous avons utilisé plusieurs technologies comme le montre l'image ci-dessous.

Nous avons décidé de choisir les services de Google Cloud Platform (GCP) pour plusieurs raisons :

    1. Le stockage : Le stockage se fait sur BigQuery (BQ), les requêtes dans BigQuery sont très rapides mais elles ont un prix.
    2. La simplicité : La configuration est faite par nous-mêmes, de plus les API REST de Google permettent facilement de se brancher aux tables BQ.
    3. L'automatisation : La partie Compute Engine permet de louer des VM (machines virtuelles) qui tournent en permanence. On verra par la suite que cela nous a permis d'automatiser l'application.
    4. La découverte : Nous avons pu découvrir de nouvelles technologies liées au cloud au lieu de rester sur Oracle ou d'autres technologies que nous avions déjà manipulées.
    5. La gestion des coûts : Chacune de nos actions avait un coût, que ce soit lié aux requêtes SQL ou à l'utilisation de VM. Cela nous a permis de prendre conscience des coûts des actions dans le cloud. À l'inscription sur GCP, nous avons une période d'essai et 300 euros offerts, nous n'avons donc pas eu besoin d'utiliser notre propre argent.

Pour la partie ETL (extraire, transformer et charger les données), nous avons choisi d'utiliser Python car c'est un langage que nous maîtrisons et que cela nous permet d'automatiser l'application. Pour la partie visualisation, nous avons choisi Power BI car il propose un large choix de visualisations et il peut se brancher à une base BQ. Pour donner un ordre d'idée sur le coût du projet, il s'élève à un peu moins de 4 euros en comptant seulement les coûts liés aux requêtes SQL et à l'utilisation de la VM. En comptant la location de la VM, nous avons atteint environ 20 euros. Pour réduire les dépenses au maximum, nous avons décidé de louer la VM la moins chère car nous n'avions pas besoin d'une VM très performante pour faire tourner notre script Python. Déploiement du script et ses composants

Le script Python et ses composants sont hébergés dans la VM GCP, voici les différents composants.

    1. La configuration : Il s'agit de la configuration liée au script Python comme des mots de passe ou une date de sauvegarde utilisée et changée par le script.
    2. Les logs : Les logs permettent de voir combien de films sont récupérés et de voir l'état des différents blocs de traitement.
    3. L'environnement virtuel Python : Il permet de gérer la version de Python et des différents packages utilisés par le script Python. Cela permet d'avoir une meilleure portabilité et d'exécuter le script sans avoir besoin d'installer les packages et dépendances.
    4. Le script Bash : Le script Python utilise des chemins relatifs, il est donc nécessaire de faire un script Bash pour se placer dans le bon répertoire lors de l'exécution automatique et lancer l'environnement virtuel Python avant de lancer le script. Voici une image du script Bash :
    5. Le script Python : Le script Python permet de récupérer, transformer et charger les données dans BQ.
    6. La clé du compte de service : Un compte de service GCP permet d'accéder à certains services GCP sans avoir besoin d'une connexion manuelle. Il est donc indispensable d'en avoir un si on veut un script qui tourne automatiquement. Il permet par exemple de se connecter à une table dans BQ. La clé est ce qui permet de s'identifier avec le compte de service au lieu de s'identifier manuellement.

Ces différents composants participent à l'exécution automatique du script mais il manque un élément, le déclencheur du script Bash. Pour lancer automatiquement le script Bash toutes les 4 heures, nous utilisons systemd qui est une suite logicielle qui fournit une gamme de composants système pour les systèmes d'exploitation Linux. On crée 2 scripts : Timer Unit qui permet de préciser les horaires d'exécution, et Service Unit qui permet de préciser ce qu'on souhaite exécuter. Voici une image pour illustrer cela :
